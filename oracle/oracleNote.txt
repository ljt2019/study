
1、Oracle数据库：相关的操作系统文件（即存储在计算机硬盘上的文件）集合，这些文件组织在一起，成为一个逻辑整体，即Oracle数据库。

2、	三个必须文件：1、Data files；2、Controller files；3、Redo Log files
	三个非必需文件：1、Parameter file；2、Password file；3、Archived Log files
 
3、系统默认用户：1、sys/system; sysman; scott(普通用户)

4、oracle的7个服务：
	1)、OracleVssWriterORCL:拷贝写入服务，非必需启动项
	
	2)、OracleDBConsoleorcl:基于web的控制台服务，在web端管理oracle时需要启动
	
	3)、OracleJobSchedulerORCL：oracle调用定时器服务
	
	4)、OracleMTSRecoveryService：服务端控制服务
	
	5)、OracleOraDb11g_home1ClrAgent：
	
	6)、OracleOraDb11g_home1TNSListener：数据库监听服务，在数据库远程访问时使用
	
	7)、OracleServiceORCL：数据库基础服务，必须启动项
	
5、什么是表空间：
	1)、表空间实际上是数据库上的逻辑存储结构，可以把表空间理解为在数据库中开辟一个空间，用于存放我们的数据库对象，一个数据库可以由多个表空间构成。
	
6、创建表空间：
	1)【create tablespace 表空间名 datafile '表空间的存储文件名.dbf' size 10m;】
	2)【create temporary tablespace 临时表空间名 tempfile '临时表空间的存储文件名.dbf' size 10m;】

7、创建用户(指定表空间和临时表空间，不然会默认为系统的表空间)：
	【create user 用户名 identified by 密码 default tablespace 表空间名 temporary tablespace 临时表空间名;】

	例如：create user linjt identified by 123456 default tablespace test_tablespace temporary tablespace temptest_tablespace;

8、查看所创建的表空间的存放位置：
	select file_name from dba_data_files where tablespace_name = 'TEST1_TABLESPACE';
	select file_name from dba_temp_files where tablespace_name = 'TEMPTEST_TABLESPACE';
	
9、查看创建的用户：【select username from dba_users;】

10、给创建的用户授权：【grant 权限 to 用户名】 例如：grant connect to linjt;
	
11、三种标准的角色（权限）：
	1、CONNECT(连接角色)：只可以登陆Oracle，不可以创建实体和数据库结构
	2、RESOURCE(资源角色)：可以创建实体，但不可以创建数据库结构
	3、DBA(数据管理员角色)：系统最高权限，只有DBA才可以创建数据库结构
	
12、链接用户:【connect 用户名/密码;】   例如：conn sys/123456 as sysdba

13、更改用户名：【alter user 用户名 identified by 新密码;】

14、锁定某用户：【alter user 用户名 account lock;】

15、删除用户：【drop user 用户名 cascade;】#加上cascade表示将用户连接以及其所创建的东西全部删除
	
16、什么事角色：即一组权限的集合；用户可以给角色赋予指定的权限，然后将角色赋给相应的用户。

17、对于普通用户授予connect和resource权限，DBA授予dba权限
